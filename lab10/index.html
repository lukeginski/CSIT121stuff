What Is RDF 1.1 Turtle?

RDF is a standart that is developed by the WWW Consortium, and stands for Resource Description Framework. The WWW Consortium is also known as the W3C, which is what creates what I use to validate all my files. RDF is a way of describing things on the web, it also describes relationships in a format that computers can understand. RDF was created in the 90s. Just because it was created then doesn't mean its old, it has been consistently getting updated and its newest version is 1.1 (there is a 1.2 in development right now). The core of RDF is that its supposed to give a structure that any system can read or use. Understanding RDF is important because it helps out busninesses with sharing data.

Turtle is a specific way of creating RDF data because its name is short for Terse RDF Triple Language. The reason turtle was created was to make it easier for humans to use RDF data. In February of 2014, the W3C made Turtle an official recommendation meaning that it is fully supported as a web standard in modern times. Since it is officially a web standard, people can rely on turtle to be consistentlyused across different programming languages and tools. When the W3C backed turtle, it was a big reason why it was widely adopted across the web.

The basic building block of any RDF document is called a "triple", which is a three-part statement made up of a subject, a predicate, and an object. It is basically like a simple sentence where the subject is what to describle, the predicate is the property/relationship, and the object is the value. For example, you could use a "triple" to say that a person has a name. Each part of a triple is identified by a URI. A URI is a Uniform Resource Identifier, and these are used so that every concept is globally unique and cannot be confused with another. This backbone of a structure allows you to build up very complex knowledge networks from just these small three-part statements.
